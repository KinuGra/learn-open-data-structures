# 第3章 / SLList（singly-linked list）

## 1. 連結リストとは

ノード（データ + `next`）をポインタでつなげた列。`head`・`tail`・`n` を保持する。

## 2. 配列との一番の違い

- **配列**：`i` 指定が速い（O(1)）／端や途中の操作は**シフト**が要る
- **連結リスト**：`i` 指定が遅い（O(n)）／**端**や**ノード参照 u 経由**の操作が速い（O(1)）

> 「配列はインデックスに強い、連結リストはポインタに強い」

## 3. SLList の O(1) 操作（定理3.1）

| 操作           | 場所                    |
| -------------- | ----------------------- |
| `push` / `pop` | 先頭                    |
| `add`          | 末尾（`tail` のおかげ） |
| `remove`       | 先頭                    |

→ Stack と FIFO Queue は完璧。

## 4. SLList の弱点

- **末尾削除は O(n)**（前ノードに戻れない）
- → Deque を作るなら **DLList（双方向）** が必要

## 5. 計算量早見表

| 操作                  | ArrayDeque     | SLList                    | DLList         |
| --------------------- | -------------- | ------------------------- | -------------- |
| get(i) / set(i)       | **O(1)**       | O(n)                      | O(n)           |
| 先頭・末尾 add/remove | O(1) 償却      | O(1)（末尾削除のみ O(n)） | **O(1)**       |
| add(i) / remove(i)    | O(min(i, n−i)) | O(i)                      | O(min(i, n−i)) |
| ノード参照 u 経由     | 概念なし       | △（直後挿入のみ）         | **O(1)**       |

## 6. 一言

- 「**`tail` を持つだけで末尾追加が O(n) → O(1)**」
- 「途中操作は **配列＝シフト O(n−i)**、**連結リスト＝たどる O(i)**」
- 「連結リストの真価は**ノード参照経由 O(1)**。代表例が **LRU キャッシュ（DLList + HashMap）**」

## 7. コーディングテスト頻出

リスト反転 / 循環検出（Floyd）/ 中央ノード（速い遅い2ポインタ）/ 2リストマージ（ダミーヘッド）/ **LRU キャッシュ**
